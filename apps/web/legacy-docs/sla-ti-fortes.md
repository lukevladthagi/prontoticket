# SLAs de TI - Sistema Fortes

Este documento define os tempos de SLA para o sistema Fortes (Folha de Pagamento e RH) no setor de TI.

## Mapeamento de Prioridades

### Incidentes (Problemas Técnicos)

**P2 - 2 horas (120 minutos)**
- Fortes - Não funciona

**P2 - 4 horas (240 minutos)**
- Fortes - Resetar senha de usuário

**P2 - 5 horas (300 minutos)**
- Fortes - Instalar
- Fortes - Configurar
- Fortes - Atualizar

### Requisições (Solicitações de Serviço)

**P3 - 24 horas (1440 minutos)**
- Fortes - Criar usuário
- Fortes - Excluir usuário
- Fortes - Ajustar usuário

## Configuração no Sistema

Os itens foram cadastrados na subcategoria "Fortes" (ID: 160) com os tempos de SLA documentados.

Para aplicar estes SLAs:
1. Problemas críticos (2h): usar tipo "Incidente" com prioridade P2
2. Reset de senha (4h): usar tipo "Incidente" com prioridade P2
3. Instalação e configuração (5h): usar tipo "Incidente" com prioridade P2
4. Gerenciamento de usuários (24h): usar tipo "Requisição" com prioridade P3
5. O sistema calculará automaticamente o prazo baseado no tipo e prioridade selecionados
