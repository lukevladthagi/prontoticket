# SLAs de TI - Sistema G4

Este documento define os tempos de SLA para o sistema G4 (Controle de Acesso) no setor de TI.

## Mapeamento de Prioridades

### Incidentes (Problemas Técnicos)

**P2 - 2 horas (120 minutos)**
- G4 - Configurar

**P2 - 4 horas (240 minutos)**
- G4 - Criar usuário
- G4 - Alterar usuário
- G4 - Excluir usuário

**P2 - 5 horas (300 minutos)**
- G4 - Não funciona

### Requisições (Solicitações de Serviço)

**P3 - 24 horas (1440 minutos)**
- G4 - Sem acesso

## Configuração no Sistema

Os itens foram cadastrados na subcategoria "G4" (ID: 176) com os tempos de SLA documentados.

Para aplicar estes SLAs:
1. Configuração (2h): usar tipo "Incidente" com prioridade P2
2. Gerenciamento de usuários (4h): usar tipo "Incidente" com prioridade P2
3. Sistema não funciona (5h): usar tipo "Incidente" com prioridade P2
4. Problemas de acesso (24h): usar tipo "Requisição" com prioridade P3
5. O sistema calculará automaticamente o prazo baseado no tipo e prioridade selecionados
